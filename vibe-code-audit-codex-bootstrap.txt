FULL CONVERSATION CONTEXT (DETAILED SUMMARY)

This project originated from the goal of creating a 100/10 “vibe code
audit” skill. The focus is auditing software built by multiple
developers working in parallel, leading to duplication, inconsistent
abstractions, and architectural drift.

Key design decisions from the discussion:

1.  The audit must be catalogue-first.
2.  It must prioritize semantic duplication (not just text duplication).
3.  It must produce refactor bundles, not laundry lists.
4.  It must be language-agnostic but optimized for Rust +
    React/TypeScript.
5.  It must use structural analysis + semantic retrieval together.

We explored tools and selected:

-   llmcc for multi-depth structural graphs (crate/module/symbol +
    PageRank).
-   agentroot for hybrid retrieval (BM25 + vector embeddings) with
    AST-aware chunking.

We decided:

V1: - The skill directly runs llmcc + agentroot. - Generates audit_index
artifacts. - Produces a structured audit report. - Uses heuristic
duplication clustering (not exhaustive).

V2: - Build Rust CLI wrapper “vibeindex”. - Formalize artifact
generation and clustering. - Skill becomes artifact consumer only.

Quality bar agreed upon: - No lint noise. - No speculative findings. -
High-confidence evidence-based output. - Prioritized refactor bundles. -
Emphasis on consolidation and architectural clarity.

Core audit phases:

1.  Indexing
2.  System mapping
3.  Pattern inconsistency scan
4.  Semantic duplication discovery
5.  Architecture boundary analysis
6.  Risk audit
7.  Refactor bundle synthesis
8.  Prioritized roadmap

Primary optimization target: Messy, organically grown Rust backend +
React frontend repositories.

The implementation must be deterministic, repeatable, and safe
(read-only).
